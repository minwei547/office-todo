## 一、项目概述

**名称**：办公室共享待办清单（office-todo）
**类型**：团队协作待办事项平台，支持实时同步、树形层级、私信、通知
**线上地址**：https://office-todo.pages.dev （Cloudflare Pages 托管）
**安卓 APK**：Capacitor 壳，直接加载线上 URL，前端更新无需重装 APK

## 二、技术栈

- **前端**：React 18 + Vite 6 + TypeScript 6 + Zustand 5 + Tailwind CSS 3
- **后端**：Supabase（PostgreSQL + Storage + Edge Functions），前端 supabase-js 直连，无自建 server
- **移动端**：Capacitor 6（安卓），加载线上 URL
- **通知**：LocalNotifications 插件（原生 App）+ Web Push（VAPID + Service Worker，浏览器）
- **实时同步**：Supabase Realtime（WebSocket）
- **设计系统**：马卡龙风格，主色 #4a7a68（深薄荷绿），背景 #fcfcfe（奶白）

- `src/store/todoStore.ts` — Zustand 全局状态：任务/团队/成员/私信，含所有 action（addTask/moveTask/swapTaskOrder/indentTask/outdentTask 等）

### 5. 升级操作无效修复（重要 bug）
- **根因**：outdentTask 用 sortOrder + 0.5（小数），但数据库 sortOrder 是 INTEGER，PostgreSQL 拒绝小数 → api 抛错 → 回滚 → "没反应"
- **修复**：indentTask/outdentTask 的新 sortOrder 改为 Math.max(同级) + 1（整数）
- 注意：两个函数命名 indent=降级(增加缩进)、outdent=升级(减少缩进)，注释已修正

### 6. 详情页子项同步
- SubTasksSection 用 useMemo 依赖 allTasks，数据订阅响应式，无需改动

### 7. 部署脚本

## 六、部署方式

**平台**：Cloudflare Pages，项目名 office-todo，连接 GitHub main 分支自动部署
**触发**：push 到 main 自动触发 `npm run build` → dist → 上线
**强制触发**：空提交 `git commit --allow-empty` 推送（项目历史多次使用）
**手动兜底**：`npm run deploy` 生成 ZIP，Cloudflare 后台 Upload assets 拖拽上传
**APK**：无需重装，前端更新后自动拿到新页面（除非改原生配置）
**远程仓库**：
- GitHub: https://github.com/minwei547/office-todo.git
- Gitee: https://gitee.com/hejin135/office-todo.git

## 七、数据模型要点

- tasks 表：parentId（TEXT，null 为顶级）、sortOrder（INTEGER，同级排序）
- 权限：队长(owner)可改所有任务，队员只能改分配给自己的任务
- moveTask 含循环引用检测（防止移到自己后代下）
